The Comark DevTools is an interactive playground that lets you write Markdown, inspect the parsed AST, and see the rendered output — all in real-time.

:a[Open DevTools]{href="/embed/devtools" target="_blank" .text-primary .underline}

---

## Overview

The DevTools interface is divided into three resizable panes:

| Pane | Description |
|------|-------------|
| **Markdown Input** | Write or paste Comark content |
| **AST Output** | View the parsed Comark AST, frontmatter, or raw JSON |
| **Preview** | See the live rendered output |

All three panes update in real-time as you type, with a short debounce (300ms) for performance.

---

## Markdown Input

The left pane is a text editor where you write Comark content. It supports the full [Comark syntax](/syntax/markdown) including:

- Standard Markdown (headings, lists, emphasis, links, code blocks, tables)
- Frontmatter (`---` delimited YAML)
- Component syntax (`::component{props}`)
- Task lists

Use the **Reset** button to restore the default example content.

---

## AST Output

The center pane displays the parser output with three tabs:

### AST Tab

Shows the `nodes` array from the parsed `ComarkTree`. Each node follows the [Comark AST format](/syntax/comark-ast):

```typescript
// Element: [tag, attributes, ...children]
['h1', { id: 'hello' }, 'Hello World']

// Text: plain string
'Some text content'
```

### Frontmatter Tab

Displays the parsed YAML frontmatter and metadata:

```json
{
  "frontmatter": {
    "title": "Hello Comark",
    "description": "A Comark playground"
  },
  "meta": {}
}
```

### Raw JSON Tab

Shows the complete `ComarkTree` object as-is, including `nodes`, `frontmatter`, and `meta`.

Use the **Copy** button to copy the current tab's JSON output to your clipboard.

---

## Preview

The right pane renders the parsed AST using the [`ComarkRenderer`](/rendering/vue) Vue component. This gives you a live preview of how your content will look when rendered.

The header shows two metrics:

- **Nodes** — Total number of nodes (elements + text) in the AST
- **Parse time** — How long parsing took in milliseconds

---

## Parser Options

The DevTools uses the following parse configuration:

```typescript
import { parse } from 'comark'

const result = await parse(markdown, {
  autoClose: true,   // Auto-close incomplete syntax (useful while typing)
  autoUnwrap: true,  // Remove unnecessary <p> wrappers
})
```

- [`autoClose`](/api/auto-close) ensures that partially typed markdown (unclosed bold, incomplete components) is handled gracefully
- `autoUnwrap` removes unnecessary paragraph wrappers from single-element containers

## Use Cases

The DevTools is useful for:

- **Learning Comark syntax** — See how markdown maps to the Comark AST
- **Debugging components** — Verify that component syntax parses correctly
- **Inspecting frontmatter** — Check that YAML frontmatter is parsed as expected
- **Performance testing** — Monitor parse times for large documents
- **Sharing examples** — Copy AST output to include in bug reports or discussions
