## Overview

The `comark/string` module provides framework-free rendering of Comark AST to HTML or Markdown strings. This is useful for server-side rendering, static site generation, RSS feeds, emails, or any context where you don't need Vue or React.

```typescript
import { parse } from 'comark'
import { renderHTML, renderMarkdown } from 'comark/string'

const { body, data } = parse('# Hello **World**')

const html = renderHTML(body)
// "<h1 id=\"hello-world\">Hello <strong>World</strong></h1>"

const markdown = renderMarkdown(body)
// "# Hello **World**"
```

---

## `renderHTML(tree, options?)`

Converts a Comark AST to an HTML string. Optionally accepts custom component renderers and data.

```typescript
import { parse } from 'comark'
import { renderHTML } from 'comark/string'

const { body } = parse(`
# Getting Started

This is a **bold** statement with a [link](https://example.com).

- Item 1
- Item 2
`)

const html = renderHTML(body)
```

Output:

```html
<h1 id="getting-started">Getting Started</h1>
<p>This is a <strong>bold</strong> statement with a <a href="https://example.com">link</a>.</p>
<ul>
<li>Item 1</li>
<li>Item 2</li>
</ul>
```

### Custom Components

When your content uses Comark components (like `::alert`), you can provide custom render functions to control their HTML output:

```typescript
import { parse } from 'comark'
import { renderHTML } from 'comark/string'

const { body } = parse(`
::alert{type="warning"}
This is a warning message!
::
`)

const html = renderHTML(body, {
  components: {
    alert: ([tag, attrs, ...children], { render }) => {
      return `<div class="alert alert-${attrs.type}" role="alert">${render(children)}</div>`
    }
  }
})
// <div class="alert alert-warning" role="alert"><p>This is a warning message!</p></div>
```

Each component render function receives:
- The full `ComarkElement` tuple `[tag, attributes, ...children]`
- A context object with:
  - `render(children: ComarkNode[])` — renders the element's children to HTML
  - `data` — frontmatter data passed via `options.data`

### With Frontmatter Data

Pass frontmatter data so component renderers can access it:

```typescript
const { body, data } = parse(`---
siteName: My Blog
---

::header
Welcome!
::
`)

const html = renderHTML(body, {
  data,
  components: {
    header: ([tag, attrs, ...children], { children, data }) => {
      return `<header><h1>${data?.siteName}</h1>${render(children)}</header>`
    }
  }
})
```

### With Syntax Highlighting

Use `parseAsync` with the `highlight` option to get syntax-highlighted code blocks in the HTML output:

```typescript
import { parseAsync } from 'comark'
import { renderHTML } from 'comark/string'

const { body } = await parseAsync('```js\nconsole.log("hi")\n```', {
  highlight: {
    themes: { light: 'github-light', dark: 'github-dark' }
  }
})

const html = renderHTML(body)
```

## `renderMarkdown(tree, data?)`

Converts a Comark AST back to Markdown. Optionally includes frontmatter data.

```typescript
import { parse } from 'comark'
import { renderMarkdown } from 'comark/string'

const source = `---
title: Hello
---

# Hello **World**

::alert{type="info"}
This is an alert.
::
`

const { body, data } = parse(source)

const markdown = renderMarkdown(body, data)
```

Output:

```yaml
---
title: Hello
---

# Hello **World**

::alert{type="info"}
This is an alert.
::
```

This is useful for normalizing content, programmatic transformations, or roundtrip testing.

## Use Cases

### Static Site Generation

```typescript
import { readFile, writeFile } from 'node:fs/promises'
import { parseAsync } from 'comark'
import { renderHTML } from 'comark/string'

async function buildPage(filePath: string) {
  const source = await readFile(filePath, 'utf-8')
  const { body, data } = await parseAsync(source, {
    highlight: true
  })

  const html = renderHTML(body)

  await writeFile('out/index.html', `
    <!DOCTYPE html>
    <html>
      <head><title>${data.title}</title></head>
      <body>${html}</body>
    </html>
  `)
}
```

### RSS Feed

```typescript
import { parse } from 'comark'
import { renderHTML } from 'comark/string'

function generateRSSItem(source: string) {
  const { body, data } = parse(source)
  return `
    <item>
      <title>${data.title}</title>
      <description><![CDATA[${renderHTML(body)}]]></description>
    </item>
  `
}
```

### API Response

```typescript
import { parse } from 'comark'
import { renderHTML } from 'comark/string'

function handleRequest(markdownContent: string) {
  const { body, data } = parse(markdownContent)

  return {
    frontmatter: data,
    html: renderHTML(body)
  }
}
```
